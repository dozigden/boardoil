# Third-Party Notices (BoardOil.Web)

This document describes how third-party licence compliance is handled for the frontend.

## Source of Truth

For runtime npm dependencies, runtime NuGet dependencies, and bundled third-party assets, the authoritative records are generated files:

- `BoardOil.Web/THIRD_PARTY_LICENSES/MANIFEST.json`
- `BoardOil.Web/THIRD_PARTY_LICENSES/*.txt`
- `BoardOil.Web/THIRD_PARTY_LICENSES/UNRESOLVED.md`

`MANIFEST.json` defines package/version/licence source mapping (including ecosystem and source type metadata).  
This notice file intentionally avoids duplicating package-by-package data.

## Generation Process

Licence files are synced using:

- `npm run sync:third-party-licences`

The sync includes:

- runtime npm dependencies listed in `package.json` `dependencies`
- runtime NuGet dependencies resolved from `BoardOil.Api/packages.lock.json`
- bundled third-party assets configured in the sync script (for example Montserrat OFL text sourced from `src/assets/fonts/montserrat/OFL-1.1.txt`)

The public runtime mirror is generated by the same sync command and also during:

- `npm run build`

For web runtime access, mirrored copies are published to:

- `BoardOil.Web/public/third-party-licenses/`

These mirrored files are generated runtime artefacts and are not source-controlled.
The source-controlled records remain in `BoardOil.Web/THIRD_PARTY_LICENSES/`.

## Manual Overrides

If an npm dependency does not ship a standalone licence file in `node_modules`, provide a manual file in:

- `BoardOil.Web/THIRD_PARTY_LICENSES_MANUAL`

If a NuGet package cannot be resolved from local cache metadata, provide a manual file in:

- `BoardOil.Web/THIRD_PARTY_LICENSES_MANUAL_NUGET`

Expected filenames are documented/generated by the sync script and unresolved entries are reported in:

- `BoardOil.Web/THIRD_PARTY_LICENSES/UNRESOLVED.md`
